Challenges for People with Cognitive Disabilities

  1. There is often a need for human, on demand help
  2. Users can easily get lost. Since help often appears in another window or within a subsection of a page, it may not be clear how to close the help portion of the site and continue with the regular workflow.
  3. It may be difficult for some users to remember and apply the help they receive

Types of Language Support

Restricted Vocabulary

Language may be restricted to a given vocabulary or a subset of that vocabulary. When words are outside the known language, an explanation should be provided using known words. Specific jargon can be added if known to the user.

Use one of the following:

  1. Restricted vocabulary such as Globish
  2. Restricted vocabulary such as Globish 100 most common words
  3. Restricted vocabulary such as Globish 500 most common words and Easy Web Jargon lexicon.

Usage

Use any or all of the following:

  1. Literal language only
  2. Active voicing whenever possible
  3. Number free content. Avoid using numbers or provide alternatives for the use of numbers, text for numbers and numerical concepts.
  4. Content in small chunks (short sentences, paragraphs etc.)

Graphical Support

Use one of the following:

  1. Symbolic images as support (pictograms or icons) with the option to select a preferred set
  2. Realistic images (pictures or drawing) as support
  3. Either Symbolic images (pictograms or icons) or Realistic images (pictures or drawing) as support

Types of Symbols and Pictures

Use one of the following:

  1. Use a given symbol or picture (set strong preference)
  2. Hide content that does not have symbol or picture support

Frequency of Symbols and Pictures

How many symbols does the user prefer? Uuse judiciously depending on context and appropriateness for content.

Use one of the following:

  1. Support symbols and pictures as much as possible
  2. Use support symbols and pictures moderately (One per sentence)
  3. Use support symbols and pictures sparingly (One per paragraph)

Help

Use any or all of the following:

  1. The option for tooltips
  2. The option for a side panel
  3. The option for a side panel that can be opened when a user clicks on a help icon
  4. The option for links to "more information" whenever possible
  5. The option to display definitions in a side panel
  6. The option for human help whenever possible

Notes

  • Help should be easy to find and displayed on demand
  • Once a user has finished using a help mechanism it should be clear how to navigate back to the primary content
  • Help content is subject to the same requirements as other content
    • For example, content should not be overwhelming, confusing or distracting
    • Content should be fully accessibile (Is there a conflict here?)
  • How are different gestures handled? For example, mobile has touch gestures.
  • Mechanisms for showing and hiding tooltips and other methods of help
    • Consider concerns with tooltips as touch devices do not have a hover state
