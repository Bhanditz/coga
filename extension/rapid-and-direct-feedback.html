Provide rapid and direct feedback

The success or failure of every user initiated action is clearly indicated to the user by visual, programmatically determinable, rapid feedback. Audio feedback should be a user selectable option.

Suggestion for Priority Level: AA

Principle and Guideline

Principle 3, Guideline 3.3

Description

Applications should continuously provide easily recognizable success or failure feedback with every user action. This information supports those who have Aphasia, Dementia, Dyslexia, or those who acquire cognitive disabilities as they age. It also helps anyone with impaired short term memory remember what they just did.

For example:

  • After a step in a multi-step task is completed the breadcrumbs display a tick or check mark nest to that step's name and if applicable, the title or name of the next step is readily apparent.
  • After a button is clicked it should look depressed, and if it is a toggle button, the state should be programmatically determinable.
  • After a form is submitted or an email is sent, feedback communicating what just happened such as "your application was submitted, thank you" or "your email was sent" is provided.

Benefits

Overt indication of the result steming from a user action helps people with a variety of cognitive disabilities understand that their action occured (e.g. the click did something), prevents uncertainty or doubt regarding the outcome, and helps them remember what they just did. User action feedback during a multi-step task can also assist people with attention or short term cognitive disabilities avoid inadvertantly leaving the task by reminding them that they are in a process and where in the process they currently are.

Testability

Procedure

  1. Trigger every user initiated action and visually inspect the screen to determine if the resulting content provides a rapid and clear indication of success or failure.
  2. If the user initiated action is part of a multi-step process, visually inspect the screen to confirm the resulting content informs the user which steps have been completed and which step they are on in the process.
  3. Trigger every user initiated action with a screen reader running to determine if the resulting content announces a rapid and clear indication of success or failure.
  4. If the user initiated action is part of a multi-step process, with a screen reader running confirm the resulting content announces to the user which steps have been completed and which step they are on in the process.

Expected Results

  • All checks above are true

Techniques

  • All existing techniques for 3.3.1
  • Using aria-pressed to provide state feedback for a toggle button
  • Providing a confirmation message when an email is successfully sent or a form is successfully submitted
  • Using a process indicator element (i.e. breadcrumbs) to communicate completed and current steps in a multi-step process
  • Providing visible and programmatically determinable information to indicate a new password satisfies the security requirement
