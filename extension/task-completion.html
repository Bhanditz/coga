SC Shortname

Task completion

SC Text

Task completion: Successfully completing tasks does rely on users memorizing information presented in the current, or previous, user-interaction dialog steps.

Note that the requirements on understandable language must also apply and the requirement on clear structure and relationships for the the a clear and unambiguous separation of menu items, which must be level A in this scope. (This may be addressed by reiterating these requirement here to increase the conformance.)

Exception: This success criterion does not apply if memorizing or otherwise mentally processing information is the primary purpose of the dialog step (e.g. a step in a game which deliberately tests the player’s memory skills).

Description

In multi-step user-interaction dialogs such as voice-menu systems, there is a risk that a user will encounter a barrier that prevents them from completing a step and, as a result, that prevents them from achieving whatever they wished to achieve. Unlike visually presented menus, which can be examined at the user’s leisure, the options in a voice menu are presented serially and often have to be held in working memory until a user is able to decide which of the options best meets their goals. The intent is make these sytems useable for people with low working memory.

 

Benefits

Without this SC many people can not use the application at all. See Voice Menu Systems issue paper for the full description of this issue and how it stops many people from using the services that often critical. Many people can not make doctors appointments etc by themselves, get there water turned back on etc.... This may be partly responsible for the reduced life expectancy of people with learning and cognitive disabilities.

The benefit of this sc is to ensure that no user is prevented from completing a user-interaction dialog because they have limited abilities to process information stored in working memory. Systems that do rely on user memorization will cause people significant stress, time spent repeatedly listening to the same voice menu and a need to resort to techniques such as writing down the options (if they have the ability to write things down). Such problems could cause unacceptable delays and possibly failure to access what could be vitally important services (e.g. emergency services). Real-life examples where failing this success criterion impacts people include inability to get a telephone line re-connected, getting urgent access to a doctor, ....


User interaction dialogs in which completion actions have a one-to-one mapping to simple discrete options may meet this SC. Examples of user-interaction dialogs that do not depend on user memorization are ones where a voice-menu system has an alternative visual menu system or ones where there is easy access to a human operator who can help the user to achieve their goal.

Testability

Test option 1: Check if one of the user authentication methods offered conforms to the sufficient techniques

 

Test option 2:

  • Step1. Identify all the actions that complete a user interaction dialog step
  • Step 2. Identify if the choice of the appropriate completion action might be dependent on information acquired from more than one serially presented chunk of information including short term memory such as remembering a number. And the menu items use simple terms and have a one-to-one mapping to simple discrete options
  • Step 3. Identify if there is an alternative way to complete the dialog step that does not rely on memorized information.

Acceptable  outcomes for test option 2:

No to step 2
OR

Yes to steps 2 and 3


 

 

Techniques

 

Techniques to include:

  • Method 1: User interaction dialogs in which
    1. completion actions have a one-to-one mapping to simple discrete options and
    2. the option and described before a number is associated with it and
    3. clear language is used as conforms to the SC X.X (understandable language)
  • Method 2. A user-interaction dialog where there is easy access to a human operator who can help the user to achieve their goal via using the standard 0 for an operator from any point
  • Method 3. Human help is offered at any step that does not conform to method 1 and
    • the option for human help is described before a number is associated with it and
    • clear language is used to for the human help option that conforms to the SC X.X (understandable language)
  • Advisory technique : Cue the user to write something down something that may be useful for the user at a later point and giving them time to do so.

 

A voice-menu system offers an alternative visual menu system.
