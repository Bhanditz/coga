What Principle and Guideline the SC falls within.

Principle 3, Guideline 3.2 - Predictable
Update to 3.2.5

Description

Any content, settings or functionality which changes unexpectedly, without user initiation can result in significant barriers for users with cognitive disabilities. Unexpected changes in any of these areas can result in loss of focus, anxiety, or confusion in understanding or using a user interface. Examples include but are not limited to:

  • Automatic launching of new windows or pop-ups
  • Submission of forms through mechanisms other than a button that is clearly labeled using simple language to submit the form
  • Slideshows without a mechanism for pausing
  • A mobile Website that disables pinch zoom which is a default setting in the browser
  • Audio or video that starts playing after page load without a mechanism for pausing
  • Live news updates without a mechanism for pausing or requesting updates

Benefits

This Success Criterion give users with cognitive disabilities more control over how Websites behave and display information enabling them to make choices that fit their individual needs.

Initiating changes only when requested by a user is particularly helpful for:

  • Focus and attention related disabilities
  • Users with low executive function
  • Memory related disabilities
  • Users with anxiety disorders

Testability

For slide shows, audio or video

  • Content does not autoplay
  • A mechanism for stoping or pausing and playing is present

For intermittent content updates such as news feeds or embedded social media updates

  • A mechanism for requesting an update is present
  • A mechanism for pausing an update is present

Techniques

  • A "pause" button for slideshows, video and audio
  • A "request update" and "pause update" button for news feeds or embedded social media updates
  • Setting the autoplay attribute on an embedded Yoytube video to "0"
