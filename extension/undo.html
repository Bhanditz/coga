SC Shortname: Undo

SC Text

Undo: Provide users the ability to undo an action and to correct mistakes

  • A user can go back steps in a process via a clearly labeled action.
  • The user can repair information via a clearly labeled action and get back to the place they were at, in a clearly labelled action without unwanted loss of data.

Suggestion for Priority Level:

AA

Related Glossary additions or changes

None

What Principle and Guideline the SC falls within.

Principle 3, Guideline 3.3.3: Error Suggestion

Description

Improving options to avoid, undo and/or correct errors is important for all users. Users should be able to edit information after it has been submitted without loss of data or should be presented with a form of confirmation prior to submission.

Benefits

Those who have difficulties accurately entering information or perceiving errors will benefit from the chance to check what has been submitted or undo an incorrect action. The benefits of having data remaining intact, even if submitted in error, allows for correction where short term memory is poor and for those with learning disabilities. Undo can provide increased confidence to continue to interact online. It allows all users, but especially those with dyslexia, re-check data entered and make necessary change if errors have occurred. Undo can simplify actions and reduce fear of failure
