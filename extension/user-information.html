What Principle and Guideline the SC falls within.

Suggested priciple is 2,

and we suggest changing guideline 2.3 from

  • Do not design content in a way that is known to cause seizures.

to

  •  Do not design content in a way that is known to harm the user.

 

 

 

Description and Benefits

 

Don’t store user personal information that could be used to harm a user.


For example storing information that suggest that a user user has dementia may make a target for scams. Storing information that suggest that a user user has an intellectual disability may make a target for predators.

For example, a predatory company would send requests of money saying “you haven’t made your donation” yet despite having made one.


It is vital that user stay safe.

Another consideration is that many of our users do have weak executive function and are less likely to identify risks correctly.



The benefit of this sc is to keep users safe while on line

OR

Yes to all three steps

 

 

Techniques

  1. For personlisation information: Using functional requirements for personalization that do not suggest that the  users is disabled.
  2. Using the aproved technique for data security for sensitive data in your jurisdiction
  3. Obtaining approval by a  Helsinki Committee  for Human Rights which is a an ethics commitess (such as used in the EU)
  4. Failure technique to be added of identifying a user to a code that is used mainly for people with cognitive disabilities.
